# Часть II. Кодирование категориальных признаков (C-4 пункт «а»)

## 2.1. Зачем кодировать?

Большинство алгоритмов МО (кроме деревьев в чистом виде, и то с оговорками) работают только с числами. Нужно превратить строки ('wild', 'domestic') в числа.

## 2.2. Виды кодирования

### **One-Hot Encoding (OHE)**

Для каждой категории создаётся **отдельный бинарный столбец** (0/1).

Если столбец `type` имеет значения `wild` и `domestic`, то после OHE:
| type | type_wild | type_domestic |
|---|---|---|
| wild | 1 | 0 |
| domestic | 0 | 1 |

### **Проблема мультиколлинеарности (drop_first)**

После OHE столбцы связаны: `type_wild + type_domestic = 1` всегда. Это создаёт **линейную зависимость** → мультиколлинеарность, которая мешает линейным моделям (но не очень страшна деревьям).

**Решение:** удалить один столбец-категорию из каждого закодированного признака. Это делается через параметр `drop='first'` (или `drop_first=True` в pandas).

После «drop first»:
| type | type_wild |
|---|---|
| wild | 1 |
| domestic | 0 |

Информация не теряется — если `type_wild = 0`, значит автоматически `domestic`.

**Правило для подсчёта:** если у категориального столбца было `k` уникальных значений, после OHE с drop_first останется **`k − 1` столбец**.

### **Label Encoding**

Категории → числа (0, 1, 2, ...). Подходит только для **деревьев** (или порядковых категорий). Для линейных моделей плохо: создаёт ложную «упорядоченность».

```python
from sklearn.preprocessing import LabelEncoder
df['col_enc'] = LabelEncoder().fit_transform(df['col'])
```

### **Target Encoding (Mean Encoding)**

Категория заменяется средним значением целевой переменной в этой категории. Мощный, но опасный (утечка) метод. Используют с регуляризацией / smoothing.

## 2.3. OHE в pandas

```python
# Самый простой способ
df_encoded = pd.get_dummies(df, columns=['type', 'group', 'education'], drop_first=True)

# По всем категориальным сразу (object и category)
df_encoded = pd.get_dummies(df, drop_first=True)

# Тип получаемых столбцов
pd.get_dummies(df, columns=['col'], drop_first=True, dtype=int)  # явно int (вместо bool)
```

**Важно:** `drop_first=True` это и есть «избавление от мультиколлинеарности».

**Как считать руками:** для каждого категориального столбца с k категорий → `k − 1` новых. Сумма по всем столбцам — итог.

Если в столбце `type` 2 категории — 1 новый столбец.
Если в `group` 3 категории — 2 новых.
И т.д. Сумма = искомое.

**Параметры DecisionTreeClassifier (полезно знать):**
| Параметр | Описание | Default |
|---|---|---|
| `criterion` | 'gini' или 'entropy' (или 'log_loss') | 'gini' |
| `max_depth` | макс. глубина | None (без ограничения) |
| `min_samples_split` | минимум объектов для разделения | 2 |
| `min_samples_leaf` | минимум объектов в листе | 1 |
| `max_features` | сколько признаков рассматривать на сплит | None |
| `random_state` | seed | None |

## 3.4. Кросс-валидация — основной способ оценки

```python
from sklearn.model_selection import cross_val_score

scores = cross_val_score(model, X, y, cv=3, scoring='roc_auc')
# cv=3 — 3 фолда
# scoring — метрика
mean_score = scores.mean()
print(round(mean_score, 1))
```

**Параметр `scoring`** — какие бывают:

| Задача | scoring |
|---|---|
| Классификация | `'accuracy'`, `'roc_auc'`, `'f1'`, `'f1_macro'`, `'precision'`, `'recall'`, `'neg_log_loss'` |
| Регрессия | `'neg_mean_squared_error'`, `'neg_mean_absolute_error'`, `'r2'`, `'neg_root_mean_squared_error'` |

**Внимание:** sklearn использует **negative** для метрик, которые надо минимизировать (MSE, MAE), потому что `cross_val_score` максимизирует scoring. Чтобы получить настоящую MSE, бери модуль: `-scores.mean()`.

## 3.6. Обычное обучение и оценка (без CV)

```python
model.fit(X_train, y_train)

# Предсказание вероятностей (для ROC-AUC, log-loss)
y_proba = model.predict_proba(X_val)[:, 1]  # вероятность класса 1

# Метрики
from sklearn.metrics import roc_auc_score, accuracy_score, f1_score
acc = accuracy_score(y_val, y_pred)

f1  = f1_score(y_val, y_pred)
```

**Важно:** `roc_auc_score` принимает **вероятности**, а не предсказанные классы! Это типичная ошибка.

## 3.7. KFold и stratified

```python
from sklearn.model_selection import KFold, StratifiedKFold

# Обычный KFold
kf = KFold(n_splits=3, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=kf, scoring='roc_auc')

# Stratified — сохраняет баланс классов в фолдах (для классификации)
skf = StratifiedKFold(n_splits=3, shuffle=True, random_state=42)
scores = cross_val_score(model, X, y, cv=skf, scoring='roc_auc')

# По умолчанию: cross_val_score для классификации использует StratifiedKFold с cv=K без перемешивания
```

**Важно:** если просто пишешь `cv=3`, sklearn для классификации применит **StratifiedKFold** без перемешивания. Результат может отличаться от `KFold(n_splits=3, shuffle=True, random_state=...)`.

# Часть VII. Полезные приёмы и продвинутые техники

## 7.1. Pipeline и ColumnTransformer

Для аккуратного «обучения сразу с препроцессингом»:

```python
from sklearn.pipeline import Pipeline
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder, StandardScaler

cat_cols = ['type', 'group', 'education', 'meal', 'preparation course']
num_cols = ['score-1', 'score-2', 'score-3']  # если бы были числовые

preprocessor = ColumnTransformer([
    ('cat', OneHotEncoder(drop='first', handle_unknown='ignore'), cat_cols),
    ('num', StandardScaler(), num_cols)
])

pipeline = Pipeline([
    ('prep', preprocessor),
    ('model', GradientBoostingClassifier(random_state=42))
])

pipeline.fit(X_train, y_train)
y_pred_proba = pipeline.predict_proba(X_test)[:, 1]
```

**Преимущества:**
- Препроцессинг автоматически применяется к train и test.
- Нет утечки данных при CV.
- Можно прокинуть в GridSearchCV целиком.

## 7.2. Масштабирование признаков

Нужно для: KNN, линейных моделей, SVM, нейросетей. НЕ нужно для деревьев/RF/бустингов.

```python
from sklearn.preprocessing import StandardScaler, MinMaxScaler

scaler = StandardScaler()  # (x - mean) / std
X_train_scaled = scaler.fit_transform(X_train)
X_test_scaled = scaler.transform(X_test)  # ← transform, не fit_transform!

# MinMaxScaler — в диапазон [0, 1]
scaler = MinMaxScaler()
```

**Принцип:** `fit` только на train, потом `transform` на test (иначе утечка).

# Часть IX. Полный шаблон финального решения для C-5

```python
import pandas as pd
import numpy as np
from sklearn.model_selection import cross_val_score, GridSearchCV, StratifiedKFold
from sklearn.preprocessing import StandardScaler
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier, GradientBoostingClassifier
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import roc_auc_score

# === 1. Чтение и предобработка ===
df_train = pd.read_csv('Data_train.csv')
df_test  = pd.read_csv('Data_test.csv')

# Заполнить пропуски (как в C-2)
for col in df_train.columns:
    if df_train[col].isna().any() or df_test[col].isna().any():
        if df_train[col].dtype == 'object':
            df_train[col] = df_train[col].fillna('Unknown')
            df_test[col]  = df_test[col].fillna('Unknown')
        else:
            m = df_train[col].mean()
            df_train[col] = df_train[col].fillna(m)
            df_test[col]  = df_test[col].fillna(m)

# Создать целевую
df_train['Pass'] = ((df_train['score-1'] > 50) &
                    (df_train['score-2'] > 50) &
                    (df_train['score-3'] > 50)).astype(int)
y = df_train['Pass']

# === 2. Feature engineering ===
df_train['cat_bio'] = '(' + df_train['type'] + ', ' + df_train['group'] + ')'
df_test['cat_bio']  = '(' + df_test['type']  + ', ' + df_test['group']  + ')'

cat_cols = ['cat_bio', 'type', 'group', 'education', 'meal', 'preparation course']

# OHE
df_train_ohe = pd.get_dummies(df_train[cat_cols], drop_first=True, dtype=int)
df_test_ohe  = pd.get_dummies(df_test[cat_cols],  drop_first=True, dtype=int)
df_test_ohe  = df_test_ohe.reindex(columns=df_train_ohe.columns, fill_value=0)

X = df_train_ohe
X_test = df_test_ohe

# === 3. Сравниваем модели по CV ===
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)

candidates = {
    'logreg': LogisticRegression(max_iter=2000, random_state=42),
    'rf':     RandomForestClassifier(n_estimators=300, random_state=42),
    'gb':     GradientBoostingClassifier(n_estimators=300, random_state=42)
}
for name, m in candidates.items():
    s = cross_val_score(m, X, y, cv=cv, scoring='roc_auc', n_jobs=-1)
    print(f"{name}: {s.mean():.4f} ± {s.std():.4f}")

# === 4. Тюним лучшую (например, GB) ===
param_grid = {
    'n_estimators': [100, 200, 300, 500],
    'max_depth': [2, 3, 4, 5],
    'learning_rate': [0.05, 0.1]
}
gs = GridSearchCV(
    GradientBoostingClassifier(random_state=42),
    param_grid, cv=cv, scoring='roc_auc', n_jobs=-1
)
gs.fit(X, y)
print(gs.best_params_, gs.best_score_)

best = gs.best_estimator_

# === 5. Финальное обучение и предсказание ===
best.fit(X, y)
proba = best.predict_proba(X_test)[:, 1]

# === 6. Сохранение ===
pd.Series(proba).to_csv('result.txt', index=False, header=False)
```

# Часть X. Чек-лист и подводные камни

## Главные ошибки

❌ **`fit_transform` на test** — это утечка. Только `transform`.

❌ **OHE без `drop_first`**, когда задача просит избавиться от мультиколлинеарности.

❌ **`predict()` вместо `predict_proba()` для AUC** — AUC посчитается некорректно.

❌ **Test и train с разными столбцами после OHE** — нужно `reindex(columns=...)` для выравнивания.

❌ **Утечка target в признаки** — например, забыли убрать `Pass` из X.

❌ **Заполнение пропусков по test или по объединению** — нужно по train.

❌ **Не учитывать stratify в задаче классификации** — баланс классов важен.

❌ **Сохранение `predict_proba(...)` без `[:, 1]`** — попадёт двумерный массив.

❌ **Round не на тот разряд**: «округлите до десятых» = `round(x, 1)`, «до сотых» = `round(x, 2)`.

❌ **Забыть `random_state`** — результаты не воспроизводятся.

## Рабочий чек-лист

✅ Заполнил пропуски (одинаково в train и test, по train).
✅ Создал целевую переменную.
✅ Все категориальные закодированы OHE с drop_first.
✅ X_test имеет те же столбцы, что и X_train (по порядку!).
✅ Для CV использую stratified для классификации.
✅ Использую `predict_proba` для AUC.
✅ Округлил ответ как просят.
✅ Файл сохранён в правильном формате.

|---|---|

| CV с ROC-AUC, 3 фолда | `cross_val_score(model, X, y, cv=3, scoring='roc_auc')` |

| Лучший скор | `gs.best_score_` |
