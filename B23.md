# B-2 и B-3. Алгоритмы машинного обучения — полная база

B-2 проверяет **любой** алгоритм МО (часто линейный или ансамбли), B-3 — конкретно **нелинейный** (KNN, деревья, SVM, KMeans). Разберу всё детально, с типовыми задачами и расчётами руками.

---

# Часть I. Линейные модели

## 1. Линейная регрессия

### Модель

$$\hat{y} = w_0 + w_1 x_1 + w_2 x_2 + \dots + w_n x_n = w^T x$$

где $w_0$ — свободный член (intercept, bias), $w_1, ..., w_n$ — коэффициенты при признаках.

В матричном виде: $\hat{y} = Xw$, где X — матрица «объект × признак» с добавленным столбцом из единиц для $w_0$.

### Функция потерь — MSE

$$L(w) = \frac{1}{n}\sum_{i=1}^{n}(y_i - w^T x_i)^2 = \frac{1}{n}\|y - Xw\|^2$$

### Аналитическое решение (Normal Equation)

$$w^* = (X^T X)^{-1} X^T y$$

**Когда применимо:**
- $X^T X$ должна быть обратимой (нет точной мультиколлинеарности).
- Для большого числа признаков медленно ($O(n^3)$).

**Численный путь:** градиентный спуск.

### Градиент

$$\nabla_w L = -\frac{2}{n} X^T(y - Xw)$$

Шаг градиентного спуска:
$$w \leftarrow w - \eta \nabla_w L = w + \frac{2\eta}{n} X^T(y - Xw)$$

### Типовая задача B-2: «найти веса по нормальному уравнению»

**Дано:** X (с единицами в первом столбце), y. Найти $w = (X^TX)^{-1}X^Ty$.

**Пример:**
$X = \begin{pmatrix} 1 & 1 \\ 1 & 2 \\ 1 & 3 \end{pmatrix}$, $y = \begin{pmatrix} 2 \\ 3 \\ 5 \end{pmatrix}$

1. $X^TX = \begin{pmatrix} 3 & 6 \\ 6 & 14 \end{pmatrix}$
2. $\det = 3·14 - 6·6 = 42 - 36 = 6$
3. $(X^TX)^{-1} = \frac{1}{6}\begin{pmatrix} 14 & -6 \\ -6 & 3 \end{pmatrix}$
4. $X^Ty = \begin{pmatrix} 2+3+5 \\ 2+6+15 \end{pmatrix} = \begin{pmatrix} 10 \\ 23 \end{pmatrix}$
5. $w = \frac{1}{6}\begin{pmatrix} 14·10 - 6·23 \\ -6·10 + 3·23 \end{pmatrix} = \frac{1}{6}\begin{pmatrix} 2 \\ 9 \end{pmatrix} = \begin{pmatrix} 1/3 \\ 3/2 \end{pmatrix}$

Итог: $\hat{y} = 1/3 + 1.5·x$.

## 2. Регуляризованные линейные модели

### Ridge (L2)

$$L(w) = \|y - Xw\|^2 + \lambda \|w\|_2^2 = \|y - Xw\|^2 + \lambda \sum w_j^2$$

**Решение:**
$$w^* = (X^TX + \lambda I)^{-1}X^Ty$$

- λ → 0: обычная регрессия.
- λ → ∞: все веса → 0.
- Гиперпараметр λ контролирует силу регуляризации.

**Важно:** регуляризация применяется только к $w_1, ..., w_n$, НЕ к свободному члену $w_0$.

### Lasso (L1)

$$L(w) = \|y - Xw\|^2 + \lambda \sum |w_j|$$

- Нет замкнутой формы — решается итерационно (координатный спуск, проксимальные методы).
- Зануляет часть весов → **отбор признаков**.

### ElasticNet

$$L(w) = \|y - Xw\|^2 + \lambda_1 \sum |w_j| + \lambda_2 \sum w_j^2$$

Компромисс L1 + L2.

### Зачем масштабировать признаки

Если один признак измеряется в тысячах, а другой в долях единицы — регуляризация будет неравномерно их штрафовать. Перед Ridge/Lasso/GD **обязательно** делать `StandardScaler`.

## 3. Логистическая регрессия

### Модель

Для бинарной классификации:

$$p(y=1 | x) = \sigma(w^T x) = \frac{1}{1 + e^{-w^T x}}$$

- Сама модель — линейная по $w^T x$ (линейная граница разделения).
- Сигмоида превращает любое число в вероятность ∈ (0, 1).

### Решающее правило

Класс 1, если $p > 0.5$ (т.е. $w^T x > 0$); иначе класс 0.

Порог 0.5 можно менять — это влияет на trade-off между Precision и Recall.

### Функция потерь — log-loss

$$L(w) = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i \log p_i + (1-y_i)\log(1-p_i)\right]$$

где $p_i = \sigma(w^T x_i)$.

### Многоклассовая (Softmax регрессия)

$$p(y=k | x) = \frac{e^{w_k^T x}}{\sum_j e^{w_j^T x}}$$

Функция потерь — categorical cross-entropy.

### Типовая задача B-2: «вычислить вероятность и предсказание»

**Дано:** w = [−1, 2, 0.5], x = [1, 1.5, −2] (первый элемент — bias).
$w^T x = -1·1 + 2·1.5 + 0.5·(-2) = -1 + 3 - 1 = 1$
$p = \sigma(1) = 1/(1+e^{-1}) ≈ 1/(1+0.368) ≈ 0.731$
Так как p > 0.5 → класс 1.

## 4. Градиентный спуск — детально

### Обновление

$$w \leftarrow w - \eta \nabla L(w)$$

### Параметры

- **η (learning rate, скорость обучения):**
  - Слишком большой → расходимость, осцилляции.
  - Слишком маленький → медленная сходимость, может застрять.
  - Часто используют убывающее η (learning rate decay) или адаптивные оптимизаторы.

### Виды

| Тип | Сколько объектов на шаг | Плюс | Минус |
|---|---|---|---|
| Batch GD | Все n | Точный градиент | Медленно, много памяти |
| SGD | 1 | Быстро, шум помогает выйти из локальных минимумов | Очень шумно |
| Mini-batch | k (32, 64, 128) | Баланс | Стандарт |

### Сходимость

- Выпуклые задачи (линейная регрессия с MSE, лог-регрессия с log-loss): GD сходится к глобальному минимуму.
- Невыпуклые (нейросети): к локальному минимуму или седловой точке.

### Типовая задача B-2: «один шаг градиентного спуска»

**Дано:** линейная регрессия, $w = [0, 0]$, $\eta = 0.1$, объекты:
$x = [(1,1), (1,2), (1,3)]$, $y = [2, 3, 5]$.

Предсказания: $\hat{y} = [0, 0, 0]$.
Остатки $(y - \hat{y}) = [2, 3, 5]$.

Градиент $-\frac{2}{n}X^T(y - Xw) = -\frac{2}{3}\begin{pmatrix} 1+1+1 \\ 1·2 + 2·3 + 3·5 \end{pmatrix}$ 

Wait — пусть $r = y - Xw = [2,3,5]$. Тогда $X^T r = \begin{pmatrix} 2+3+5 \\ 1·2+2·3+3·5 \end{pmatrix} = \begin{pmatrix} 10 \\ 23 \end{pmatrix}$.

$\nabla L = -\frac{2}{3}\begin{pmatrix} 10 \\ 23 \end{pmatrix} ≈ \begin{pmatrix} -6.67 \\ -15.33 \end{pmatrix}$

$w_{new} = w - \eta \nabla L = (0,0) - 0.1·(-6.67, -15.33) = (0.667, 1.533)$

# Часть II. Нелинейные модели (B-3 — основной фокус)

## 5. K-ближайших соседей (KNN)

### Идея

«Скажи мне, кто твои соседи, и я скажу, кто ты».
- Для классификации: смотрим на K ближайших объектов из обучающей выборки → голосование большинством.
- Для регрессии: среднее (или взвешенное среднее) целевых значений K соседей.

### Алгоритм

1. Для нового объекта $x$ посчитать расстояние до **всех** объектов из train.
2. Найти K ближайших.
3. Классификация: большинство голосов; Регрессия: среднее $y$.

### Метрики расстояния

- **Евклидова (L2):** $d(x,y) = \sqrt{\sum(x_i - y_i)^2}$ — по умолчанию.
- **Манхэттенская (L1):** $d(x,y) = \sum |x_i - y_i|$ — устойчивее к выбросам.
- **Минковского:** $d(x,y) = (\sum |x_i - y_i|^p)^{1/p}$ — обобщение.
- **Косинусная:** $1 - \frac{x \cdot y}{\|x\| \|y\|}$ — для текстов, эмбеддингов.
- **Хэмминга:** для бинарных/категориальных признаков.

### Гиперпараметры

- **K (число соседей):**
  - K=1 — переобучение (точная подгонка под train).
  - Большой K — недообучение (всё «смазывается»).
  - Подбирается кросс-валидацией.
  - Часто для бинарной классификации берут **нечётное** K, чтобы избежать ничьих.
- **Веса:** uniform (все соседи равны) vs distance (ближе сосед — больше голос).
- **Метрика расстояния.**

### Важно

- **Масштабирование признаков обязательно** (иначе один признак с большим масштабом доминирует).
- **«Проклятие размерности»**: при большом числе признаков расстояния становятся почти одинаковыми → KNN деградирует.
- KNN ничего не обучает (lazy learning): всё считается на этапе предсказания → медленный inference.

### Типовая задача B-3: «предскажите класс/значение KNN»

**Дано:**

| x₁ | x₂ | y |
|---|---|---|
| 1 | 1 | A |
| 2 | 2 | A |
| 3 | 3 | B |
| 6 | 5 | B |
| 7 | 7 | B |

Новый объект: $x = (3, 2)$. K = 3, метрика — евклидова.

Расстояния:
- до (1,1): $\sqrt{4+1} = \sqrt{5} ≈ 2.24$
- до (2,2): $\sqrt{1+0} = 1$
- до (3,3): $\sqrt{0+1} = 1$
- до (6,5): $\sqrt{9+9} = \sqrt{18} ≈ 4.24$
- до (7,7): $\sqrt{16+25} = \sqrt{41} ≈ 6.40$

3 ближайших: (2,2)→A, (3,3)→B, (1,1)→A.
Голоса: A=2, B=1 → **класс A**.

Для регрессии (если y — числа): среднее y трёх соседей.

## 6. Решающие деревья (Decision Trees)

### Идея

Древовидная структура «вопрос-ответ»: в каждом узле — условие на признаке (например, $x_3 \leq 5.2$), листы содержат предсказания.

### Алгоритм построения (CART)

1. **Корень:** все данные.
2. Перебираем все признаки и пороги, выбираем сплит, **максимально уменьшающий критерий неоднородности**.
3. Делим данные на 2 части; рекурсивно строим деревья для каждой.
4. **Останавливаемся**, когда:
   - Глубина = max_depth.
   - В узле < min_samples_split объектов.
   - Узел «чистый» (все объекты одного класса).
   - Или другой stopping criterion.

### Критерии для классификации

**Энтропия:**
$$H(S) = -\sum_{k=1}^{K} p_k \log_2 p_k$$
где $p_k$ — доля класса k в узле S.

- H = 0, если все объекты одного класса.
- H = $\log_2 K$ (максимум) при равномерном распределении.

**Information Gain:**
$$IG = H(parent) - \sum_{child} \frac{|S_{child}|}{|S_{parent}|} H(S_{child})$$

Выбираем сплит с максимальным IG.

**Индекс Джини:**
$$Gini(S) = 1 - \sum_{k=1}^{K} p_k^2$$

- Gini = 0 — чистый узел.
- Близко к энтропии, но проще в вычислении (нет log).
- По умолчанию в sklearn — Gini.

### Критерии для регрессии

**MSE в узле:**
$$L(S) = \frac{1}{|S|}\sum_{i \in S}(y_i - \bar{y}_S)^2$$

Выбирают сплит, минимизирующий взвешенную сумму MSE детей.

В листе предсказание = среднее $y$ объектов в этом листе.

### Гиперпараметры

- **max_depth** — максимальная глубина.
- **min_samples_split** — минимум объектов для разделения узла.
- **min_samples_leaf** — минимум объектов в листе.
- **max_features** — сколько признаков рассматривать при сплите.
- **criterion** — gini / entropy / mse / mae.

### Свойства

✓ Не требует масштабирования признаков.
✓ Обрабатывает категориальные и числовые признаки.
✓ Легко интерпретируется (можно визуализировать).
✓ Нелинейный, строит «прямоугольные» области решения.

✗ Сильно склонны к переобучению (особенно без ограничений глубины).
✗ Нестабильны: малые изменения данных → разные деревья.

### Типовая задача B-3: «энтропия / Джини / IG в узле»

**Дано:** узел содержит 10 объектов: 6 класса A, 4 класса B.

- $p_A = 0.6, p_B = 0.4$
- **Энтропия:** $H = -(0.6 \log_2 0.6 + 0.4 \log_2 0.4)$
  - $0.6·\log_2 0.6 = 0.6·(-0.737) = -0.442$
  - $0.4·\log_2 0.4 = 0.4·(-1.322) = -0.529$
  - $H = -(-0.442 - 0.529) = 0.971$
- **Gini:** $1 - (0.6^2 + 0.4^2) = 1 - 0.52 = 0.48$

**Сплит:** делим на 2 узла:
- Левый: 5 объектов (4A, 1B) → $p_A = 0.8$
  - $H_{left} = -(0.8\log_2 0.8 + 0.2\log_2 0.2) = -(-0.258 - 0.464) = 0.722$
- Правый: 5 объектов (2A, 3B) → $p_A = 0.4$
  - $H_{right} = -(0.4\log_2 0.4 + 0.6\log_2 0.6) = 0.971$

**IG = $H_{parent} - \frac{5}{10}H_{left} - \frac{5}{10}H_{right}$**
$= 0.971 - 0.5·0.722 - 0.5·0.971 = 0.971 - 0.361 - 0.486 = 0.124$

### Типовая задача B-3: «предсказание дерева для нового объекта»

Тебе дают дерево (картинку или правила), новый объект → пройди по дереву от корня до листа, верни ответ.

Например:
```
if x1 <= 5:
    if x2 <= 3: → класс A
    else: → класс B
else: → класс C
```
Объект (4, 5): x1≤5, x2>3 → **B**.

## 7. Random Forest (Случайный лес)

### Идея

Это **бэггинг** над решающими деревьями + случайный выбор признаков на каждом сплите.

### Алгоритм

1. Сделать M bootstrap-подвыборок размера n из train (с возвращением).
2. На каждой обучить глубокое дерево; **в каждом узле** рассматривать случайное подмножество признаков размера $\sqrt{n}$ (классификация) или $n/3$ (регрессия).
3. Предсказание ансамбля:
   - Классификация: голосование (большинство).
   - Регрессия: среднее.

### Зачем случайный выбор признаков?

Без него все деревья были бы похожи (на «сильных» признаках). Случайность делает их **более разнообразными**, и усреднение сильнее снижает variance.

### OOB error (Out-of-Bag)

Объекты, не попавшие в bootstrap-подвыборку конкретного дерева, можно использовать как «тест» для этого дерева. Усреднение даёт OOB-оценку → бесплатная оценка обобщающей способности без отдельной кросс-валидации.

### Гиперпараметры

- **n_estimators** — число деревьев. Больше = лучше, но медленнее. Обычно 100–500.
- **max_depth, min_samples_leaf** и пр. — как у деревьев.
- **max_features** — сколько признаков на сплит.

### Свойства

✓ Очень устойчив к переобучению (благодаря бэггингу).
✓ Хорошо работает «из коробки».
✓ Естественно даёт **feature importance**.
✓ Не требует масштабирования.

✗ Медленнее и тяжелее, чем одно дерево.
✗ Хуже интерпретируется.

## 8. Gradient Boosting (Градиентный бустинг)

### Идея

Строим последовательно модели, каждая исправляет ошибки предыдущих. Каждая следующая модель приближает антиградиент функции потерь.

### Алгоритм (для регрессии с MSE)

1. Инициализируем $F_0(x) = $ среднее $y$.
2. На шаге m:
   - Вычисляем остатки: $r_i = y_i - F_{m-1}(x_i)$.
   - Обучаем слабую модель $h_m$ предсказывать остатки.
   - $F_m(x) = F_{m-1}(x) + \eta \cdot h_m(x)$.
3. Повторяем M раз.

Для общей функции потерь $h_m$ предсказывает **антиградиент** $-\frac{\partial L}{\partial F(x_i)}$.

### Гиперпараметры

- **n_estimators (M)** — число итераций. Слишком много → переобучение.
- **learning_rate (η)** — обычно 0.01–0.3. Меньше η + больше M = обычно лучше.
- **max_depth** базового дерева — 3–8 типично (слабые модели!).
- **subsample** — доля выборки на каждой итерации (stochastic boosting).

### Современные реализации

- **XGBoost** — оптимизированный, с регуляризацией, многопоточный.
- **LightGBM** — leaf-wise growth, быстрее на больших данных.
- **CatBoost** — лучше с категориальными признаками, ordered boosting (борьба с утечкой target).

### Свойства

✓ Часто лучшие результаты на табличных данных.
✓ Гибкий — работает с разными функциями потерь.

✗ Склонен к переобучению (надо настраивать параметры).
✗ Чувствительнее к шуму в данных, чем RF.

### Сравнение с Random Forest

| | Random Forest | Gradient Boosting |
|---|---|---|
| Параллельность | Да | Нет (последовательно) |
| Уменьшает | Variance | Bias |
| Базовые деревья | Глубокие | Мелкие (3–8) |
| Чувствительность к параметрам | Низкая | Высокая |
| Переобучение | Редко | Часто (надо настраивать) |
| Качество out-of-the-box | Хорошее | Часто лучше при тюнинге |

### Типовая задача B-2: «один шаг бустинга»

Дано: 3 объекта с $y = [10, 20, 30]$.
- $F_0(x) = $ среднее = 20.
- Остатки: $r = [-10, 0, 10]$.
- Дерево $h_1$ предсказывает остатки, например, $h_1(x) = [-9, 1, 9]$.
- При $\eta = 0.1$: $F_1(x) = 20 + 0.1·[-9, 1, 9] = [19.1, 20.1, 20.9]$.

## 9. Метод опорных векторов (SVM)

### Идея

Найти гиперплоскость, максимально разделяющую классы (с максимальным «зазором»).

### Линейная SVM

$$y \cdot (w^T x + b) \geq 1 - \xi_i, \quad \xi_i \geq 0$$

Минимизируем $\frac{1}{2}\|w\|^2 + C\sum \xi_i$.

- C — параметр регуляризации (как 1/λ): больше C = меньше регуляризация (жёстче подгонка).
- $\xi_i$ — slack-переменные (нарушение margin).

### Kernel trick — нелинейность

Подменяем скалярное произведение $x^T z$ на kernel $K(x, z)$:
- **Линейное:** $K(x,z) = x^T z$.
- **Полиномиальное:** $K(x,z) = (x^T z + c)^d$.
- **RBF (gaussian):** $K(x,z) = \exp(-\gamma \|x-z\|^2)$ — самый популярный.

Этим достигается «нелинейная разделимость» без явного перехода в многомерное пространство.

### Гиперпараметры

- **C** — регуляризация.
- **kernel.**
- **γ** (для RBF) — обратная ширина «горки». Большая γ — модель ближе к каждому объекту (склонна к переобучению).

### Свойства

✓ Эффективен в высокой размерности.
✓ Хорош для текстов (вместе с TF-IDF) и сложных границ.

✗ Плохо масштабируется на большие данные ($O(n^2)$–$O(n^3)$).
✗ Чувствителен к масштабу — нужен StandardScaler.

---

## 10. Наивный Байес (Naive Bayes)

### Идея

Применяем формулу Байеса с **наивным** предположением о независимости признаков:

$$P(y | x) \propto P(y) \prod_j P(x_j | y)$$

### Виды

- **Gaussian NB** — для непрерывных признаков (предполагает нормальность внутри класса).
- **Multinomial NB** — для счётных признаков (тексты, BoW).
- **Bernoulli NB** — для бинарных признаков.

### Свойства

✓ Очень быстрый. Хорошо работает на текстах.
✓ Мало данных — норм.

✗ Наивное предположение часто нарушается; вероятности могут быть плохо калиброваны.

# Часть III. Кластеризация (нелинейные методы)

## 11. K-Means

### Алгоритм

1. Задаём K.
2. Инициализируем K центров (случайно или K-means++).
3. **Шаг назначения:** каждую точку относим к ближайшему центру.
4. **Шаг обновления:** новые центры = средние точек кластеров.
5. Повторяем 3–4 до сходимости (центры почти не двигаются).

### Цель

$$\text{Inertia} = \sum_{k=1}^{K}\sum_{x \in C_k} \|x - \mu_k\|^2$$

K-means гарантированно уменьшает inertia на каждом шаге, но сходится в **локальный** минимум → нужно несколько случайных запусков (`n_init`).

### Гиперпараметры

- **K** — число кластеров.
- **n_init** — число случайных инициализаций (sklearn по умолчанию 10).
- **max_iter** — макс. итераций.

### Метод локтя (elbow) для выбора K

Рисуем график inertia vs K. Ищем «излом» — точку, после которой улучшение становится медленным.

### Типовая задача B-3: «один шаг K-means»

**Дано:** точки [(1,1), (2,2), (5,5), (6,6)], начальные центры $c_1 = (1,1), c_2 = (5,5)$.

Назначение:
- (1,1) → $c_1$ (расстояние 0)
- (2,2) → $c_1$ (расстояние √2 < √18)
- (5,5) → $c_2$
- (6,6) → $c_2$

Обновление:
- $c_1 = (1.5, 1.5)$
- $c_2 = (5.5, 5.5)$

## 12. DBSCAN

### Идея

Кластер — это связная область высокой плотности. Точки в разреженных областях — шум.

### Параметры

- **eps (ε)** — радиус соседства.
- **min_samples** — минимум точек в радиусе ε для того, чтобы точка считалась core.

### Типы точек

- **Core:** в радиусе ε ≥ min_samples соседей (включая саму).
- **Border:** в радиусе ε core-точки, но сама не core.
- **Noise:** не core и не border.

### Алгоритм

1. Для каждой точки определяем тип.
2. Соединяем достижимые core-точки → формируем кластер.
3. Прикрепляем border-точки к ближайшему кластеру.
4. Точки-шум не входят ни в один кластер.

### Свойства

✓ Не требует задавать K.
✓ Находит кластеры произвольной формы.
✓ Естественно работает с выбросами.

✗ Плохо при кластерах разной плотности.
✗ Сложно подобрать eps в высокой размерности.

### Типовая задача B-3: «определить типы точек DBSCAN»

**Дано:** точки и eps, min_samples. Для каждой точки посчитать число соседей в радиусе eps → определить core/border/noise.

# Часть IV. Что часто спрашивают на экзамене

### Типы вопросов

**По линейным моделям:**
- «Найдите $w$ по нормальному уравнению» (формула, аккуратно посчитать).
- «Сделайте один шаг градиентного спуска».
- «Чему равно предсказание линейной/логистической регрессии для объекта $x$?»
- «Чему равна вероятность класса 1?»

**По KNN:**
- «Найдите класс/предсказание для нового объекта при K=…»
- «Какая метрика расстояния используется».
- «Что произойдёт при K=1 / K=n?»

**По деревьям:**
- «Чему равна энтропия / Gini в узле?»
- «Чему равен information gain после сплита?»
- «По какому признаку и при каком пороге дерево выберет разделение?»
- «Какое предсказание дерева для нового объекта?»

**По бустингу:**
- «Какие остатки на первом шаге?»
- «Чему равно $F_1(x)$ после одного шага?»

**По кластеризации:**
- «Один шаг K-means: куда отнесутся точки, какие новые центры?»
- «Чему равна inertia?»
- «Какие точки — core/border/noise при заданных параметрах DBSCAN?»

---

### Лайфхаки для экзамена

1. **Внимательно читай условие** — порой просят не финальное состояние, а одно из промежуточных.
2. **Веди записи аккуратно** — формулы с матрицами легко напутать.
3. **Сверяй размерности:**
   - $X$ — (n × d), $w$ — (d,) или (d × 1), $y$ — (n,).
   - $X^T X$ — (d × d), $X^T y$ — (d,).
4. **При проверке логистической регрессии:**
   - $\sigma(0) = 0.5$
   - $\sigma$ — монотонна, $\sigma(+\infty) = 1$, $\sigma(-\infty) = 0$.
   - Класс 1 ⇔ $w^T x > 0$ ⇔ $p > 0.5$.
5. **KNN:** не забудь, что нужно посчитать **все** расстояния и упорядочить.
6. **Дерево:** $\log_2$ — это не натуральный log! На калькуляторе: $\log_2 x = \ln x / \ln 2$.
7. **K-means:** после назначения сразу пересчитывай центр как среднее координат.
8. **Бустинг:** не забывай умножать на learning rate.

---

### Маленький справочник формул

| Алгоритм | Ключевая формула |
|---|---|
| Линейная регрессия | $w^* = (X^TX)^{-1}X^Ty$ |
| Ridge | $w^* = (X^TX + \lambda I)^{-1}X^Ty$ |
| Сигмоида | $\sigma(z) = 1/(1+e^{-z})$ |
| Лог-регрессия (предсказание) | $p = \sigma(w^Tx)$ |
| Log-loss | $-\frac{1}{n}\sum[y\log p + (1-y)\log(1-p)]$ |
| Энтропия | $H = -\sum p_k \log_2 p_k$ |
| Gini | $1 - \sum p_k^2$ |
| Information Gain | $IG = H_{parent} - \sum \frac{|S_c|}{|S_p|} H_c$ |
| MSE в узле | $\frac{1}{|S|}\sum(y_i - \bar y_S)^2$ |
| Бустинг | $F_m = F_{m-1} + \eta h_m$, $h_m$ ≈ остатки |
| K-means inertia | $\sum_k \sum_{x \in C_k} \|x - \mu_k\|^2$ |
| Евклидово расстояние | $\sqrt{\sum (x_i - y_i)^2}$ |
| Сигмоида в точках | $\sigma(1)≈0.731$, $\sigma(2)≈0.881$, $\sigma(-1)≈0.269$ |
