# B-1. Расчёт метрик ошибки по формулам — полная база

Это самый важный «технический» вопрос: тебе дают данные (предсказания и истинные значения) и просят посчитать конкретную метрику. Здесь нет смысла рассуждать — нужно знать формулу и аккуратно посчитать.

---

## Часть 1. Регрессионные метрики

Везде обозначения:
- $y_i$ — истинное значение
- $\hat{y}_i$ — предсказание модели
- $n$ — число объектов
- $\bar{y}$ — среднее истинных значений

---

### 1.1. MAE (Mean Absolute Error)

$$MAE = \frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|$$

**Алгоритм:**
1. Для каждого объекта посчитать $|y_i - \hat{y}_i|$ (модуль разности).
2. Сложить всё.
3. Разделить на $n$.

**Пример:**
y = [3, −0.5, 2, 7]
ŷ = [2.5, 0.0, 2, 8]

|разности| = [0.5, 0.5, 0, 1]
Сумма = 2
MAE = 2/4 = **0.5**

---

### 1.2. MSE (Mean Squared Error)

$$MSE = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

**Алгоритм:**
1. Разности $y_i - \hat{y}_i$.
2. Возвести в квадрат.
3. Сложить.
4. Разделить на $n$.

**Пример (те же данные):**
разности: [0.5, −0.5, 0, −1]
квадраты: [0.25, 0.25, 0, 1]
сумма = 1.5
MSE = 1.5/4 = **0.375**

---

### 1.3. RMSE (Root Mean Squared Error)

$$RMSE = \sqrt{MSE} = \sqrt{\frac{1}{n}\sum(y_i - \hat{y}_i)^2}$$

**Пример:** RMSE = √0.375 ≈ **0.612**

---

### 1.4. MAPE (Mean Absolute Percentage Error)

$$MAPE = \frac{100\%}{n}\sum_{i=1}^{n}\left|\frac{y_i - \hat{y}_i}{y_i}\right|$$

Иногда без 100% (тогда это доля, не процент). **На экзамене внимательно смотри, требуется ли результат в процентах.**

**Алгоритм:**
1. Считаешь $|y_i - \hat{y}_i| / |y_i|$ для каждого объекта.
2. Усредняешь.
3. Умножаешь на 100% (если нужно).

**Пример:**
y = [10, 20, 50]
ŷ = [12, 18, 55]
доли ошибок: |2|/10 = 0.2; |2|/20 = 0.1; |5|/50 = 0.1
сумма = 0.4
среднее = 0.4/3 ≈ 0.133
MAPE = **13.33%**

**Подводные камни:**
- Если хотя бы один $y_i = 0$ → MAPE не определена.
- Несимметрична: занижение и завышение штрафуются по-разному.

---

### 1.5. SMAPE (Symmetric MAPE)

$$SMAPE = \frac{100\%}{n}\sum \frac{|y_i - \hat{y}_i|}{(|y_i| + |\hat{y}_i|)/2}$$

Решает проблему деления на 0 и асимметрии.

---

### 1.6. R² (коэффициент детерминации)

$$R^2 = 1 - \frac{\sum(y_i - \hat{y}_i)^2}{\sum(y_i - \bar{y})^2} = 1 - \frac{SS_{res}}{SS_{tot}}$$

где:
- $SS_{res} = \sum(y_i - \hat{y}_i)^2$ — сумма квадратов остатков (residuals)
- $SS_{tot} = \sum(y_i - \bar{y})^2$ — общая сумма квадратов
- $\bar{y}$ — среднее истинных значений

**Алгоритм:**
1. Найти $\bar{y}$.
2. Посчитать $SS_{res}$ (как для MSE, но без деления на n).
3. Посчитать $SS_{tot}$ (квадраты отклонений от среднего, без деления на n).
4. R² = 1 − $SS_{res}/SS_{tot}$.

**Пример:**
y = [3, 5, 2, 8, 7]
ŷ = [2.5, 5, 4, 8, 6]

$\bar{y}$ = (3+5+2+8+7)/5 = 25/5 = 5

Остатки: 0.5, 0, −2, 0, 1 → квадраты: 0.25, 0, 4, 0, 1 → SS_res = 5.25

Отклонения от среднего: −2, 0, −3, 3, 2 → квадраты: 4, 0, 9, 9, 4 → SS_tot = 26

R² = 1 − 5.25/26 ≈ 1 − 0.2019 ≈ **0.798**

**Интерпретация:**
- R² = 1 — идеально.
- R² = 0 — модель не лучше, чем предсказывать среднее.
- R² < 0 — модель хуже среднего (бывает на тесте).

---

### 1.7. Huber loss

$$L_\delta(a) = \begin{cases} \frac{1}{2}a^2, & |a| \leq \delta \\ \delta(|a| - \frac{1}{2}\delta), & |a| > \delta \end{cases}$$

где $a = y - \hat{y}$. Гибрид MSE (для маленьких ошибок) и MAE (для больших, устойчивость к выбросам).

## Часть 2. Классификационные метрики

### 2.1. Матрица ошибок (confusion matrix) — основа всего

Для бинарной классификации:

|  | Предсказан 0 | Предсказан 1 |
|---|---|---|
| Истинный 0 | **TN** (true neg) | **FP** (false pos) |
| Истинный 1 | **FN** (false neg) | **TP** (true pos) |

**Запоминалка:**
- True/False — угадали ли (T = правильно, F = ошибка).
- Positive/Negative — что предсказали.
- **FP = «ложная тревога»** (мы сказали «1», на самом деле «0»).
- **FN = «пропустили цель»** (мы сказали «0», на самом деле «1»).

**Пример** — мы строим эту таблицу:

y_true = [1, 0, 1, 1, 0, 1, 0, 0, 1, 0]
y_pred = [1, 0, 1, 0, 0, 1, 1, 0, 1, 1]

Идём по парам:
| i | y | ŷ | категория |
|---|---|---|---|
| 1 | 1 | 1 | TP |
| 2 | 0 | 0 | TN |
| 3 | 1 | 1 | TP |
| 4 | 1 | 0 | FN |
| 5 | 0 | 0 | TN |
| 6 | 1 | 1 | TP |
| 7 | 0 | 1 | FP |
| 8 | 0 | 0 | TN |
| 9 | 1 | 1 | TP |
| 10 | 0 | 1 | FP |

Подсчёт: TP = 4, TN = 3, FP = 2, FN = 1. Всего 10. ✓

Эта таблица — нам сейчас её и будем гонять во все формулы.

---

### 2.2. Accuracy

$$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$

Доля правильных ответов.

**Пример:** Accuracy = (4+3)/10 = **0.7** (или 70%).

**Подвох:** при дисбалансе классов даёт обманчиво высокое значение. Если 95% объектов — класс 0, тривиальная модель «всё 0» даст Accuracy = 0.95.

---

### 2.3. Precision (точность)

$$Precision = \frac{TP}{TP + FP}$$

«Из тех, кому мы поставили 1, сколько действительно 1».

**Пример:** Precision = 4/(4+2) = 4/6 ≈ **0.667**

---

### 2.4. Recall (полнота, sensitivity, TPR)

$$Recall = \frac{TP}{TP + FN}$$

«Из всех настоящих 1, сколько мы поймали».

**Пример:** Recall = 4/(4+1) = 4/5 = **0.8**

---

### 2.5. F1-score

$$F1 = \frac{2 \cdot Precision \cdot Recall}{Precision + Recall} = \frac{2 TP}{2 TP + FP + FN}$$

Гармоническое среднее Precision и Recall. Если хотя бы одна = 0, F1 = 0.

**Пример:** F1 = (2·0.667·0.8)/(0.667+0.8) = 1.067/1.467 ≈ **0.727**

Или по второй формуле: F1 = (2·4)/(2·4 + 2 + 1) = 8/11 ≈ **0.727**. ✓

---

### 2.6. Fβ-score

$$F_\beta = (1 + \beta^2) \cdot \frac{Precision \cdot Recall}{\beta^2 \cdot Precision + Recall}$$

- β > 1 — Recall важнее (например, F2).
- β < 1 — Precision важнее (F0.5).
- β = 1 — обычный F1.

---

### 2.7. Specificity (TNR)

$$Specificity = \frac{TN}{TN + FP}$$

Доля верно опознанных отрицательных.

**Пример:** Specificity = 3/(3+2) = **0.6**

---

### 2.8. FPR (False Positive Rate)

$$FPR = \frac{FP}{FP + TN} = 1 - Specificity$$

---

### 2.9. NPV (Negative Predictive Value)

$$NPV = \frac{TN}{TN + FN}$$

«Из тех, кому поставили 0, сколько действительно 0».

---

### 2.10. Balanced Accuracy

$$BA = \frac{Recall + Specificity}{2} = \frac{TPR + TNR}{2}$$

Полезна при дисбалансе.

---

## Часть 3. Многоклассовая классификация

Пусть K классов. Для каждого класса k можно построить «one-vs-rest» матрицу ошибок и посчитать $P_k, R_k, F1_k$.

### 3.1. Macro-averaging

Среднее по классам **без учёта размера**:
$$Precision_{macro} = \frac{1}{K}\sum_k Precision_k$$
$$F1_{macro} = \frac{1}{K}\sum_k F1_k$$

Все классы важны одинаково.

### 3.2. Weighted-averaging

Среднее по классам **с весами = доля класса в данных**:
$$F1_{weighted} = \sum_k \frac{n_k}{n} \cdot F1_k$$

### 3.3. Micro-averaging

Суммируем TP, FP, FN по всем классам, потом считаем:
$$Precision_{micro} = \frac{\sum_k TP_k}{\sum_k TP_k + \sum_k FP_k}$$

Для многоклассовой задачи micro-precision = micro-recall = micro-F1 = **Accuracy**.

**Пример (макроусреднение):**

3 класса (A, B, C). Допустим:
- F1_A = 0.8, F1_B = 0.6, F1_C = 0.4

Macro F1 = (0.8 + 0.6 + 0.4)/3 = **0.6**

Если в выборке: 50 объектов A, 30 B, 20 C (n=100):
Weighted F1 = 0.5·0.8 + 0.3·0.6 + 0.2·0.4 = 0.4 + 0.18 + 0.08 = **0.66**

## Часть 4. Метрики на основе вероятностей

### 4.1. Log-loss (Binary Cross-Entropy)

$$LogLoss = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i \log(p_i) + (1-y_i)\log(1-p_i)\right]$$

где $p_i$ — предсказанная вероятность класса 1.

**Алгоритм:**
1. Если $y_i = 1$ → берём $\log(p_i)$.
2. Если $y_i = 0$ → берём $\log(1 - p_i)$.
3. Со знаком минус усредняем.

**Пример:**
y = [1, 0, 1]
p = [0.9, 0.2, 0.6]

i=1: y=1 → log(0.9) ≈ −0.105
i=2: y=0 → log(1−0.2) = log(0.8) ≈ −0.223
i=3: y=1 → log(0.6) ≈ −0.511

Сумма ≈ −0.839
LogLoss = −(−0.839)/3 ≈ **0.280**

Чем меньше — тем лучше.

### 4.2. Categorical Cross-Entropy (многоклассовая)

$$CE = -\frac{1}{n}\sum_{i=1}^{n}\sum_{k=1}^{K} y_{ik}\log(p_{ik})$$

Где $y_{ik}$ — one-hot истинной метки.

---

### 4.3. ROC-AUC

**ROC-кривая:** TPR (Recall) vs FPR при разных порогах классификации.

**AUC** = площадь под ROC-кривой ∈ [0, 1].
- 1 — идеальный классификатор.
- 0.5 — случайный.
- < 0.5 — хуже случайного (модель «перевёрнута»).

**Интерпретация:** AUC = вероятность того, что случайно выбранный положительный объект получит больший скор, чем случайно выбранный отрицательный.

**Алгоритм расчёта вручную (на маленьком примере):**

Объекты с истинными метками и вероятностями:

| Объект | y | p |
|---|---|---|
| A | 1 | 0.9 |
| B | 0 | 0.7 |
| C | 1 | 0.6 |
| D | 0 | 0.4 |
| E | 0 | 0.2 |

Считаем пары (pos, neg):
- (A, B): 0.9 > 0.7 — победа поз → 1
- (A, D): 0.9 > 0.4 → 1
- (A, E): 0.9 > 0.2 → 1
- (C, B): 0.6 < 0.7 → 0
- (C, D): 0.6 > 0.4 → 1
- (C, E): 0.6 > 0.2 → 1

Всего пар: 2 поз × 3 нег = 6. Побед: 5.
AUC = 5/6 ≈ **0.833**

Если одинаковые скоры — за такие пары даём 0.5.

## Часть 5. Полезные приёмы для экзамена

### 5.1. Как не запутаться в TP/FN/FP/TN

Всегда задавай два вопроса о каждом объекте:
1. **Что мы предсказали** (Positive или Negative)?
2. **Правильно ли** (True или False)?

«TP» — мы сказали Positive и угадали (True). «FN» — мы сказали Negative и ошиблись.

### 5.2. Связи формул

- Precision и Recall несравнимы напрямую с Accuracy при дисбалансе.
- F1 = 0, если Precision или Recall = 0.
- F1 = 1 только если оба = 1.
- Accuracy = (TP+TN)/n, при этом FP+FN — это ошибки.

### 5.3. На экзамене

1. Сначала **построй матрицу ошибок** (если она не дана) — пройдись по всем парам (y, ŷ).
2. Сверь: TP + TN + FP + FN = n.
3. Подставь в формулу нужной метрики.
4. **Округление:** обычно до 2–4 знаков после запятой; ориентируйся на формат поля ввода.

## Часть 6. Типичные вопросы экзамена B-1 и шаблоны решения

### Тип 1. «По матрице ошибок посчитайте F1»

Дано: TP=40, TN=50, FP=10, FN=20.

```
Precision = 40/(40+10) = 0.8
Recall = 40/(40+20) = 40/60 ≈ 0.667
F1 = 2·0.8·0.667/(0.8+0.667) = 1.067/1.467 ≈ 0.727
```

Или сразу: F1 = 2·TP/(2·TP+FP+FN) = 80/(80+10+20) = 80/110 ≈ **0.727**

### Тип 2. «Даны y_true и y_pred — посчитайте Accuracy/Precision/Recall/F1»

1. Построй матрицу ошибок (как в примере выше).
2. Подставь в формулы.

### Тип 3. «Даны y и ŷ (числа) — посчитайте MAE/MSE/RMSE/R²»

1. Считай ошибки $y_i - \hat{y}_i$.
2. Применяй формулу.

### Тип 4. «Даны вероятности — посчитайте log-loss»

1. Для каждого объекта: $-y_i \log p_i - (1-y_i)\log(1-p_i)$.
2. Усредни.

### Тип 5. «Дана матрица 3×3 (многоклассовая) — посчитайте Accuracy / macro-F1»

Например:
```
            предсказано
            A  B  C
истинно A [ 10  2  3 ]
истинно B [  1 15  4 ]
истинно C [  2  3 10 ]
```

Accuracy = (10+15+10) / 50 = **0.70**

Для F1 по класс А (one-vs-rest):
- TP_A = 10 (А предсказан как А)
- FP_A = 1 + 2 = 3 (другие классы, предсказаны как А) — это столбец A без диагонали
- FN_A = 2 + 3 = 5 (А, но предсказан как другой) — строка A без диагонали
- Precision_A = 10/13, Recall_A = 10/15, F1_A = …

Аналогично для B, C. Macro-F1 = средняя F1 по классам.

## Часть 7. Полезный код (для самопроверки)

```python
from sklearn.metrics import (
    mean_absolute_error, mean_squared_error, r2_score,
    accuracy_score, precision_score, recall_score, f1_score,
    confusion_matrix, classification_report,
    roc_auc_score, log_loss
)
import numpy as np

# Регрессия
mae = mean_absolute_error(y_true, y_pred)
mse = mean_squared_error(y_true, y_pred)
rmse = np.sqrt(mse)  # или mean_squared_error(..., squared=False)
r2 = r2_score(y_true, y_pred)

# Классификация
acc = accuracy_score(y_true, y_pred)
prec = precision_score(y_true, y_pred)
rec = recall_score(y_true, y_pred)
f1 = f1_score(y_true, y_pred)
cm = confusion_matrix(y_true, y_pred)  # [[TN, FP], [FN, TP]]

# Вероятностные
auc = roc_auc_score(y_true, y_proba)
ll = log_loss(y_true, y_proba)

# Многоклассовая
f1_macro = f1_score(y_true, y_pred, average='macro')
f1_weighted = f1_score(y_true, y_pred, average='weighted')
```

**Важно:** `confusion_matrix` в sklearn возвращает массив в порядке:
```
[[TN  FP]
 [FN  TP]]
```
А некоторые источники используют другую раскладку (с TP в левом верхнем). Будь внимателен.

## Часть 8. Чек-лист перед расчётом

✅ Уточни, какой класс «положительный» (по умолчанию обычно 1).
✅ Правильно построй матрицу ошибок: TP + TN + FP + FN = n.
✅ Не путай Precision и Recall:
   - **Precision** — делитель содержит **FP** (ложные «1»).
   - **Recall** — делитель содержит **FN** (пропущенные «1»).
✅ Для регрессии — обычно ошибка $y - \hat{y}$, но в модуле/квадрате знак неважен.
✅ R² использует $\bar{y}$, не среднее предсказаний.
✅ MAPE → проверь, проценты или доля; не делить на 0.
✅ Log-loss — будь осторожен с p=0 или p=1 (часто клипуют до [ε, 1−ε]).
✅ Не забывай делить на n там, где это нужно (MAE, MSE, MAPE), но НЕ нужно в SS_res, SS_tot для R².
