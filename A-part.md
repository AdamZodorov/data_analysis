## А-1. Визуализация — полная база

### Типы графиков и когда их использовать

**Одна числовая переменная (распределение):**
- **Гистограмма** — частоты по бинам. Подходит для понимания формы распределения.
- **KDE (kernel density estimation)** — сглаженная плотность.
- **Бокс-плот** — медиана, квартили, выбросы.
- **Скрипичный (violin) plot** — бокс-плот + KDE по бокам.
- **Strip plot / swarm plot** — точечный показ всех значений.

**Одна категориальная:**
- **Bar chart (столбчатая)** — частоты категорий.
- **Pie chart** — доли (только если категорий 2–5, иначе нечитаемо).

**Две числовые:**
- **Scatter plot (точечный)** — связь, корреляция, кластеры.
- **Hexbin / 2D-гистограмма** — если точек очень много (заменяет scatter).
- **Линейный график (line plot)** — если ось X упорядочена (время, индексы).

**Категориальная + числовая:**
- **Бокс-плот / violin по группам** — сравнение распределений.
- **Bar chart со средними** + error bars.

**Две категориальные:**
- **Heatmap частот** (contingency table) или group bar chart.

**Три и более переменных:**
- **Bubble chart** — scatter, где размер точки = 3-я переменная, цвет = 4-я.
- **Heatmap** — матрица значений (например, корреляций).
- **Pair plot** — все попарные scatter-плоты.
- **Facet grid** — несколько подграфиков по группам.

**Время:**
- **Линейный график** — динамика.
- **Stacked area** — динамика долей.

### Чтение бокс-плота

```
       │
   ┬───────────┐
───┤   |       ├───   (горизонтальный)
   ┴───────────┘
       │
  усы  Q1 медиана Q3 усы   • выбросы
```

- **Коробка:** от Q1 (25-й перцентиль) до Q3 (75-й).
- **Линия внутри:** медиана.
- **IQR = Q3 − Q1.**
- **Усы:** до самой дальней точки в пределах [Q1 − 1.5·IQR, Q3 + 1.5·IQR].
- **Точки за усами:** выбросы.
- **Если медиана не в центре коробки** → распределение скошено.
- **Длинный верхний ус** → правое смещение.

### Скрипичный (violin) plot

- Ширина в каждой точке = плотность данных там.
- Часто внутри показан мини-бокс-плот.
- Лучше, чем бокс-плот, когда распределение бимодальное (два горба).

### Пузырьковый график

- Оси X, Y — две числовые переменные.
- **Размер пузырька** — третья числовая.
- **Цвет** — четвёртая (числовая или категориальная).
- Минусы: трудно сравнивать размеры на глаз.

### Heatmap

- Цвет = значение в ячейке.
- Часто для матриц корреляций (цветовая шкала от −1 до 1, дивергентная палитра).

### Типичные ошибки в визуализации

- 3D pie chart — искажение восприятия.
- Усечённая ось Y у bar chart — преувеличение различий.
- Слишком много категорий в pie chart.
- Использование цвета без значения.

## А-2. PDF, CDF — полная база

### Плотность (PDF) f(x)

Для **непрерывной** случайной величины:
- f(x) ≥ 0
- ∫f(x)dx по всей оси = 1
- P(a ≤ X ≤ b) = ∫ₐᵇ f(x)dx — **вероятность = площадь под кривой**
- f(x) сама по себе НЕ вероятность; может быть >1 (например, у равномерного на [0, 0.5] плотность = 2)
- P(X = a) = 0 для любого конкретного a у непрерывной величины

### Функция распределения (CDF) F(x)

F(x) = P(X ≤ x)
- F(−∞) = 0, F(+∞) = 1
- F монотонно не убывает
- F непрерывна справа
- Связь: F'(x) = f(x); f(x) = dF/dx
- P(a < X ≤ b) = F(b) − F(a)

### Как читать графики

**По PDF:**
- Пик(и) → мода (моды). Бимодальное распределение = 2 пика.
- Симметрия относительно вертикальной линии → среднее = медиана = мода (примерно).
- Длинный правый хвост → **правое (положительное) смещение** → среднее > медианы > моды.
- Длинный левый хвост → **левое (отрицательное) смещение** → среднее < медианы < моды.
- Ширина «горба» → разброс (дисперсия).

**По CDF:**
- Где CDF растёт круто → высокая плотность.
- Где плато → плотность ≈ 0.
- F(x) = 0.5 → медиана.
- F(x) = 0.25 → Q1, F(x) = 0.75 → Q3.
- Ступенька на CDF → дискретная величина (есть атом массы).

### Основные распределения

**Нормальное N(μ, σ²):**
- Симметрично, колоколообразное.
- 68% попадает в [μ−σ, μ+σ], 95% в [μ−2σ, μ+2σ], 99.7% в [μ−3σ, μ+3σ].
- Среднее = медиана = мода = μ.

**Равномерное U(a,b):**
- Плотность = 1/(b−a) на [a,b], 0 вне.
- Среднее = (a+b)/2, дисперсия = (b−a)²/12.

**Экспоненциальное Exp(λ):**
- f(x) = λe⁻λˣ для x ≥ 0.
- Среднее = 1/λ. Сильное правое смещение.

**Бернулли Bern(p):** 1 с вероятностью p, 0 с (1−p). E=p, D=p(1−p).

**Биномиальное Bin(n,p):** число успехов в n испытаниях. E=np, D=np(1−p).

**Пуассона Pois(λ):** число событий в фиксированный промежуток. E=D=λ.

## А-3. Теория вероятностей — углублённо

### Базовые свойства

- 0 ≤ P(A) ≤ 1
- P(Ω) = 1, P(∅) = 0
- P(Ā) = 1 − P(A)
- P(A∪B) = P(A) + P(B) − P(A∩B)
- Если A ⊆ B → P(A) ≤ P(B)
- Несовместные события: P(A∩B) = 0 → P(A∪B) = P(A) + P(B)

### Независимость

A и B независимы ⇔ P(A∩B) = P(A)·P(B)
**Важно:** «несовместность ≠ независимость». Несовместные с ненулевыми вероятностями всегда зависимы!

### Условная вероятность

P(A|B) = P(A∩B)/P(B), при P(B) > 0

Умножение: P(A∩B) = P(A|B)·P(B) = P(B|A)·P(A)

### Формула полной вероятности

Если H₁,...,Hₙ — полная группа гипотез (несовместны, в объединении дают Ω):

$$P(A) = \sum_{i=1}^{n} P(A|H_i) \cdot P(H_i)$$

### Формула Байеса

$$P(H_i|A) = \frac{P(A|H_i) \cdot P(H_i)}{P(A)} = \frac{P(A|H_i) \cdot P(H_i)}{\sum_j P(A|H_j) \cdot P(H_j)}$$

- P(Hᵢ) — априорная вероятность
- P(Hᵢ|A) — апостериорная (после наблюдения A)

### Формула включений-исключений

Для 2 событий: P(A∪B) = P(A)+P(B) − P(AB)

Для 3: P(A∪B∪C) = P(A)+P(B)+P(C) − P(AB) − P(AC) − P(BC) + P(ABC)

Для n: знаки чередуются, суммы всех 1-, 2-, ..., n-пересечений.

### Математическое ожидание

- Дискретное: E(X) = Σ xᵢ·P(X=xᵢ)
- Непрерывное: E(X) = ∫x·f(x)dx

**Свойства (всегда верны):**
- E(c) = c
- E(aX + b) = aE(X) + b
- E(X + Y) = E(X) + E(Y) — **линейность; X, Y могут быть зависимыми!**

**Только для независимых:**
- E(XY) = E(X)·E(Y)

### Дисперсия

D(X) = E[(X − E(X))²] = E(X²) − [E(X)]²

Свойства:
- D(c) = 0
- D(aX + b) = a²·D(X)
- D(X + Y) = D(X) + D(Y) + 2·Cov(X,Y)
- Для независимых: D(X+Y) = D(X) + D(Y)
- σ(X) = √D(X) — стандартное отклонение

### Ковариация и корреляция

- Cov(X,Y) = E[(X−EX)(Y−EY)] = E(XY) − E(X)E(Y)
- ρ(X,Y) = Cov(X,Y) / (σₓ·σᵧ) ∈ [−1, 1]
- Если X, Y независимы → Cov = 0 (обратное не верно!)

## А-4. Гипотезы и корреляция

### Постановка задачи

- **H₀** — нулевая гипотеза («нет эффекта», статус-кво).
- **H₁** — альтернативная (то, что мы хотим доказать).
- **Статистика критерия** — функция от выборки.
- **Критическое множество (область отклонения)** — значения статистики, при которых H₀ отвергается.

### Ошибки

|  | H₀ верна | H₀ ложна |
|---|---|---|
| Не отвергли H₀ | OK | Ошибка II рода (β) |
| Отвергли H₀ | Ошибка I рода (α) | OK (мощность) |

- **α (уровень значимости)** = P(отвергнуть H₀ | H₀ верна). Обычно 0.05, 0.01.
- **β** = P(не отвергнуть H₀ | H₁ верна).
- **Мощность = 1 − β** = P(отвергнуть H₀ | H₁ верна).

Цель: при фиксированном α максимизировать мощность.

### p-value

P-значение = вероятность получить значение статистики **такое же или более экстремальное** при условии, что H₀ верна.
- Если p < α → отвергаем H₀.
- Если p ≥ α → нет оснований отвергнуть.
- **p-value ≠ вероятность того, что H₀ верна!**

### Виды критериев

- **Двусторонние** (≠) и **односторонние** (>, <).
- t-тест (сравнение средних), z-тест, χ²-тест (категориальные), F-тест (дисперсии), Манна-Уитни (непараметрический).

### Корреляция

**Пирсон:**
- $r = \frac{\sum(x_i-\bar x)(y_i-\bar y)}{\sqrt{\sum(x_i-\bar x)^2 \sum(y_i-\bar y)^2}}$
- r ∈ [−1, 1]
- Измеряет **линейную** связь.
- r = 0 — нет линейной связи (но может быть y = x²!).

**Спирмен:** корреляция Пирсона между **рангами** значений. Ловит монотонную нелинейную связь.

**Кендалл:** на согласованных/несогласованных парах. Устойчивее на малых выборках.

**Интерпретация |r|:**
- 0.0–0.2: очень слабая
- 0.2–0.4: слабая
- 0.4–0.7: средняя
- 0.7–1.0: сильная

**Важно:** корреляция ≠ причинность. И матрица корреляций по умолчанию — Пирсон в pandas.

## А-5. Линейная алгебра — углублённо

### Векторы

- Сумма: покомпонентно.
- Умножение на скаляр: покомпонентно.
- **Скалярное произведение:** a·b = Σaᵢbᵢ = |a||b|cosθ
  - a·b = 0 ⇒ перпендикулярны.
  - cosθ = a·b / (|a||b|) — **косинусная мера сходства**, используется в NLP.
- **Норма L2:** ||a||₂ = √Σaᵢ²
- **Норма L1:** ||a||₁ = Σ|aᵢ|

### Матрицы — операции

- **Сложение:** размерности должны совпадать.
- **Умножение A(m×n)·B(n×k):** размерности по «средним» индексам должны совпадать. Результат: m×k.
  - **НЕ коммутативно:** AB ≠ BA (даже когда оба возможны).
  - Ассоциативно: (AB)C = A(BC).
  - Дистрибутивно: A(B+C) = AB + AC.
- **Транспонирование:**
  - (Aᵀ)ᵢⱼ = Aⱼᵢ
  - (A+B)ᵀ = Aᵀ + Bᵀ
  - (AB)ᵀ = BᵀAᵀ  ← **обратный порядок!**
- **След (trace):** tr(A) = Σ Aᵢᵢ. Свойство: tr(AB) = tr(BA).
- **Определитель:** для 2×2: ad−bc. Для 3×3: правило Саррюса/разложение по строке.
  - det(AB) = det(A)·det(B)
  - det(Aᵀ) = det(A)
  - Если det(A) = 0 → матрица **вырожденная** (необратимая).

### Обратная матрица

A·A⁻¹ = A⁻¹·A = I
- Существует ⇔ det(A) ≠ 0.
- (AB)⁻¹ = B⁻¹A⁻¹
- (Aᵀ)⁻¹ = (A⁻¹)ᵀ
- Для 2×2: $A^{-1} = \frac{1}{\det A}\begin{pmatrix}d & -b \\ -c & a\end{pmatrix}$

### Ранг

- Число линейно независимых строк = число линейно независимых столбцов.
- rank(A) ≤ min(m, n).
- Полный ранг — когда rank = min(m, n).

### Собственные значения и векторы

- Av = λv, v ≠ 0
- Уравнение: det(A − λI) = 0 → собственные значения.
- Сумма собственных значений = tr(A).
- Произведение = det(A).
- Для симметричных матриц: все λ вещественные, собственные векторы ортогональны.

### Матричные разложения

**Спектральное (только для квадратных):** A = QΛQ⁻¹, где Q — собственные векторы, Λ — диагональная с λ.

**SVD (Singular Value Decomposition) — для любых матриц m×n:**

$$A = U \cdot \Sigma \cdot V^T$$

- **U** (m×m) — ортогональная (столбцы — левые сингулярные векторы).
- **Σ** (m×n) — **прямоугольная "диагональная"**: на главной диагонали стоят **сингулярные числа** σ₁ ≥ σ₂ ≥ ... ≥ 0, остальные элементы = 0.
- **Vᵀ** (n×n) — ортогональная (правые сингулярные векторы как строки).
- Сингулярные числа = квадратные корни из собственных значений матрицы AᵀA.
- Число ненулевых σ = ранг A.

**Свойства SVD, важные для экзамена:**
- Σ имеет ту же форму, что и A (m × n).
- Сумма элементов матрицы Σ = сумма всех сингулярных чисел (т.к. остальные элементы — нули).
- Если матрица A умножается на скаляр c, то её сингулярные числа умножаются на |c|. То есть SVD(cA) даёт Σ' = |c|·Σ.

**Пример (как в твоей задаче):**
Если у тебя A = 10·X и тебя спрашивают про SVD матрицы 10X, то Σ_{10X} = 10·Σ_X. Сумма элементов = 10·(σ₁ + σ₂ + ... + σᵣ) для X.

В Python:
```python
import numpy as np
U, s, Vt = np.linalg.svd(10*X)
# s — одномерный массив сингулярных чисел
sum_sigma = s.sum()
```

**Применения SVD:** PCA, LSA, рекомендательные системы, сжатие изображений, низкоранговые приближения.

## А-6. Тип задачи МО — расширенно

### Главный вопрос: что мы предсказываем?

**1. Есть целевая переменная (с учителем):**
- Число → **регрессия** (цена, температура, число просмотров)
- Категория (2 класса) → **бинарная классификация** (спам, отток, болен/здоров)
- Категория (>2 классов, эксклюзивных) → **многоклассовая классификация** (цифра 0–9)
- Несколько меток одновременно → **multi-label классификация** (теги статьи)

**2. Нет целевой (без учителя):**
- Разбить на группы → **кластеризация**
- Снизить размерность → **dimensionality reduction** (PCA, t-SNE, UMAP)
- Найти редкие/необычные объекты → **anomaly detection**
- Найти зависимости → **association rules** (часто вместе покупают)

**3. Другое:**
- Рекомендации (часто = матричное разложение + регрессия/классификация).
- Временные ряды → специфический тип регрессии.
- Reinforcement Learning — обучение через взаимодействие со средой.

### Конкретные подсказки

| Формулировка | Тип |
|---|---|
| «Предсказать число X» | Регрессия |
| «Предсказать класс / категорию / да-нет» | Классификация |
| «Разделить клиентов на группы (без меток)» | Кластеризация |
| «Найти аномалии / необычное» | Anomaly detection |
| «Сжать представление» | Dimensionality reduction |
| «Найти рейтинг релевантности» | Ranking |

## А-7. Качество модели и переобучение — углублённо

### Переобучение (overfitting)

**Симптомы:**
- Высокий score на train, низкий на test/val.
- Большая разница между train и test метриками.

**Причины:**
- Слишком сложная модель (много параметров).
- Мало данных.
- Слишком долгое обучение.
- Сильные шумы в данных.
- Утечка данных (data leakage).

### Недообучение (underfitting)

**Симптомы:**
- Низкий score и на train, и на test.

**Причины:**
- Модель слишком простая.
- Слишком сильная регуляризация.
- Признаков мало или они неинформативные.

### Способы борьбы с переобучением

1. Больше данных (или аугментация).
2. Упростить модель (меньше глубина дерева, меньше нейронов).
3. **Регуляризация:**
   - **L2 (Ridge):** штраф λΣwᵢ² → веса маленькие, но не нулевые.
   - **L1 (Lasso):** штраф λΣ|wᵢ| → часть весов = 0 (отбор признаков).
   - **ElasticNet:** комбинация L1 + L2.
4. **Dropout** (для NN).
5. **Early stopping** — остановить обучение по валидационной ошибке.
6. **Кросс-валидация** — для надёжной оценки.
7. **Аугментации** (картинки, тексты).
8. **Batch normalization**, ensemble методы.

### Кросс-валидация

**K-Fold:** делим train на K частей. По очереди каждая часть = валидация, остальные = обучение. Получаем K оценок → усредняем.

**Stratified K-Fold:** сохраняет баланс классов в каждом фолде (для классификации).

**Leave-One-Out (LOO):** K = N. Очень долго, используется на малых данных.

**Time Series Split:** для временных рядов — обучаемся на прошлом, тестируем на будущем (не перемешиваем!).

### Подбор гиперпараметров

**GridSearchCV:** перебор всех комбинаций из заданной сетки. Точный, но медленный.

**RandomSearchCV:** случайный выбор N комбинаций. Эффективнее при большом пространстве.

**Bayesian optimization:** Optuna, hyperopt. Умнее, чем случайный поиск.

### Кривые обучения

- **Learning curve:** train/val score vs размер выборки.
  - Обе ошибки высокие и близкие → underfitting.
  - Train ошибка низкая, val высокая → overfitting.

2. **Токенизация** — разбить на слова/токены.
   - Простая: split по пробелам.
   - Продвинутая: nltk, spaCy, BPE/WordPiece для трансформеров.
3. **Удаление стоп-слов** (the, и, а, в, на...).
4. **Нормализация:**
   - **Стемминг** — отрезает окончания по правилам, даёт «обрубок» (бежал → беж, running → run). Быстро, грубо, может выдать несуществующее слово.
   - **Лемматизация** — приводит к словарной форме (бежал → бежать, better → good). Точно, медленнее, требует словарей.

### Векторизация текста

**Bag of Words (BoW):**
- Словарь всех слов.
- Каждый документ — вектор частот.
- Теряет порядок слов.

**TF-IDF:**
- **TF(t, d)** — частота слова t в документе d (часто: count / общее число слов в d).
- **IDF(t) = log(N / df(t))**, N — число документов, df(t) — в скольких документах встречается t.
- **TF-IDF(t, d) = TF(t, d) · IDF(t)**.
- Идея: ↑ когда слово часто в данном документе и редко в остальных.
- Слово, которое во всех документах → IDF ≈ 0 → не важно.

**Embeddings:** word2vec, GloVe, FastText, BERT-эмбеддинги. Плотные векторы фиксированной размерности; семантически близкие слова → близкие векторы.

**N-граммы:** последовательности из N подряд идущих токенов (биграммы: «машинное обучение»). Учитывают локальный порядок.

### Изображения

**Представление:**
- Чёрно-белое: матрица H×W со значениями обычно 0–255 (uint8).
- Цветное (RGB): тензор H×W×3.
- Бинарное: значения 0/1.

**Нормализация:**
- Деление на 255 → [0, 1].
- Стандартизация: (x − μ)/σ покаждому каналу (для предобученных моделей mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225] для ImageNet).

**Аугментации:**
- Геометрические: повороты, отражения, кропы, сдвиги, масштабирование.
- Цветовые: яркость, контраст, насыщенность.
- Шум, blur.
- Mixup, CutMix.

**Свёртки (CNN):**
- Применяют небольшое ядро (например, 3×3) к каждой части изображения.
- Уменьшают параметры (по сравнению с FC) и используют локальность.
- Pooling уменьшает разрешение и делает инвариантным к малым сдвигам.

## А-9, А-10. Метрики — полная база

### Регрессия

**MAE (Mean Absolute Error):**
$$MAE = \frac{1}{n}\sum|y_i - \hat y_i|$$
- Те же единицы, что и y.
- Устойчива к выбросам.
- Не дифференцируема в нуле, но это для большинства алгоритмов не проблема.

**MSE (Mean Squared Error):**
$$MSE = \frac{1}{n}\sum(y_i - \hat y_i)^2$$
- Сильно штрафует большие ошибки (квадрат).
- Чувствительна к выбросам.
- Единицы — y².

**RMSE:** √MSE. Единицы y. Часто используется как «штраф за большие промахи, но в исходных единицах».

**MAPE (Mean Absolute Percentage Error):**
$$MAPE = \frac{100\%}{n}\sum\left|\frac{y_i - \hat y_i}{y_i}\right|$$
- В процентах, интерпретируема.
- **Плохо**, если y близко к 0 (деление на маленькое число → огромная ошибка).
- Несимметрична: занижение и завышение штрафуются по-разному.

**R² (коэффициент детерминации):**
$$R^2 = 1 - \frac{\sum(y_i-\hat y_i)^2}{\sum(y_i - \bar y)^2}$$
- 1 — идеально, 0 — модель не лучше среднего, <0 — хуже среднего.
- Доля объясняемой дисперсии.

### Бинарная классификация

**Матрица ошибок:**

|  | Предсказано 0 | Предсказано 1 |
|---|---|---|
| Истинно 0 | TN | FP |
| Истинно 1 | FN | TP |

**Accuracy = (TP + TN) / (TP+TN+FP+FN)**
- Доля верных ответов.
- **Плохо при дисбалансе!** Если 99% класса 0, модель «всё нули» даёт 99% accuracy.

**Precision (точность) = TP / (TP + FP)**
- «Из тех, кого назвали положительными, сколько действительно положительные».
- Важна, когда дорого ошибиться положительным предсказанием (например, рекомендация дорогого товара).

**Recall = Sensitivity = TPR = TP / (TP + FN)**
- «Из всех настоящих положительных, сколько мы поймали».
- Важна, когда дорого пропустить (диагностика болезни, мошенники).

**F1-score = 2·P·R / (P + R)** — гармоническое среднее Precision и Recall. Если хотя бы одна низкая — F1 низкий.

**Fβ-score:** настраиваемая важность Precision vs Recall.

**Specificity (TNR) = TN / (TN + FP)** — доля верно опознанных отрицательных.

**ROC-кривая:** TPR vs FPR при разных порогах. **AUC** ∈ [0, 1]:
- 1 — идеальный классификатор.
- 0.5 — случайный.
- <0.5 — хуже случайного (можно инвертировать).
- Интерпретация: вероятность того, что случайный положительный объект получит более высокий скор, чем случайный отрицательный.

**PR-кривая** (Precision-Recall): лучше для несбалансированных классов.

**Log-loss (Cross-entropy):**
$$-\frac{1}{n}\sum[y_i \log p_i + (1-y_i)\log(1-p_i)]$$
- Использует вероятности, а не классы. Сильно штрафует уверенные неправильные ответы.

### Многоклассовая

- **Accuracy** — доля правильных.
- **Macro F1** — усреднение F1 по классам без учёта размера (все классы важны одинаково).
- **Weighted F1** — взвешенное по размеру класса.
- **Micro F1** — суммируем TP, FP, FN по всем классам и считаем общий F1.
- **Confusion matrix** — обобщение матрицы ошибок.

### Когда какую метрику использовать — таблица

| Ситуация | Метрика |
|---|---|
| Регрессия, важно среднее отклонение в исходных единицах | MAE / RMSE |
| Регрессия, есть выбросы | MAE |
| Регрессия, важно сильнее штрафовать большие промахи | MSE / RMSE |
| Регрессия, нужны проценты, y не близко к 0 | MAPE |
| Регрессия, сравнение моделей | R² |
| Классификация, классы сбалансированы | Accuracy |
| Классификация, дисбаланс | F1, ROC-AUC, PR-AUC |
| Дорого пропустить положительные (рак) | Recall |
| Дорого ложно сработать (важный спам-фильтр) | Precision |
| Ранжирование / probabilistic output | ROC-AUC, log-loss |
| Многоклассовая, важны редкие классы | Macro F1 |
| Многоклассовая, важна общая точность | Accuracy / Weighted F1 |

### Пример из твоего вопроса (А-10/B-1)

**«Предсказать число просмотров статьи»** — это **регрессия**.

Из вариантов:
- ROC-AUC — для бинарной классификации, **не подходит**.
- MSE — регрессионная метрика, **подходит**.
- F1-score — для классификации, **не подходит**.
- Accuracy — для классификации, **не подходит**.

**Ответ: MSE.**

### Интерпретация значений метрик

- **R² = 0.85** — модель объясняет 85% дисперсии целевой.
- **AUC = 0.7** — на 70% времени положительный объект будет ранжирован выше отрицательного.
- **MAE = 100 (для числа просмотров)** — в среднем ошибаемся на 100 просмотров.
- **F1 = 0.6** — компромисс между Precision и Recall на уровне 0.6.

## А-11. Линейные модели и градиентный спуск

### Линейная регрессия

$$\hat y = w_0 + w_1 x_1 + \dots + w_n x_n = w^T x$$

Минимизирует **MSE**.

**Аналитическое решение:** w = (XᵀX)⁻¹Xᵀy
- Требует обратимости XᵀX (нет точной мультиколлинеарности).
- O(n³) — медленно при большом числе признаков.

**Регуляризованные версии:**
- **Ridge (L2):** добавляет λ||w||₂². Решение: w = (XᵀX + λI)⁻¹Xᵀy.
- **Lasso (L1):** λ||w||₁. Аналитически не решается; зануляет коэффициенты.
- **ElasticNet:** комбинация.

### Логистическая регрессия

Бинарная классификация:
$$p = \sigma(w^T x), \quad \sigma(z) = \frac{1}{1 + e^{-z}}$$

Минимизирует **log-loss**.

Решающее правило: класс 1, если p > порога (по умолчанию 0.5).

**Декомпозиция уровней решения:** линейная разделяющая гиперплоскость.

### Градиентный спуск

Обновление весов:
$$w \leftarrow w - \eta \nabla L(w)$$

- **η (learning rate)** — слишком большой → расходимость, слишком маленький → медленно.
- Для невыпуклых задач может застрять в локальном минимуме.
- Для выпуклых (линейная регрессия с MSE, лог-регрессия с log-loss) сходится к глобальному минимуму.

**Виды:**
- **Batch GD:** градиент по всей выборке. Точно, но медленно.
- **SGD:** по одному примеру. Быстро, но шумно. Может «выскочить» из локальных минимумов.
- **Mini-batch:** по батчу (32, 64, 128...). Стандарт.

**Продвинутые оптимизаторы:**
- **Momentum:** учитывает «инерцию» предыдущих шагов.
- **Adam:** адаптивный learning rate, наиболее популярен.
- **RMSprop, Adagrad** и др.

### Важное для линейных моделей

- **Масштабирование признаков** (StandardScaler) обязательно при использовании GD и регуляризации.
- Признаки коррелированы → нестабильные коэффициенты (мультиколлинеарность).
- Категориальные → нужно кодировать (one-hot, target encoding).

## А-12. Композиции моделей

### Бэггинг (Bagging)

- **Bootstrap Aggregating.**
- Из N объектов берём N с возвращением → одна подвыборка. Так делаем M подвыборок.
- Обучаем M моделей независимо.
- Объединяем: среднее (регрессия) или голосование (классификация).
- **Уменьшает variance, bias почти не меняет.**
- Модели должны быть **сильными** (склонными к переобучению — например, глубокие деревья).
- Можно обучать параллельно.

**Random Forest:**
- Бэггинг над деревьями + на каждом узле случайно выбираем подмножество признаков для сплита.
- Это уменьшает корреляцию между деревьями → лучше работает.
- Обычно: для классификации √n признаков, для регрессии n/3.
- Out-of-bag (OOB) error — оценка без отдельного теста (используем те объекты, которые не попали в bootstrap).

### Бустинг (Boosting)

- Модели обучаются **последовательно**.
- Каждая следующая исправляет ошибки предыдущих.
- **Уменьшает bias.**
- Базовые модели — **слабые** (например, пеньки — деревья глубины 1–3).
- Подвержен переобучению при большом числе итераций.

**AdaBoost:** на каждой итерации увеличиваем веса неправильно классифицированных объектов.

**Gradient Boosting:** на каждой итерации новая модель обучается предсказывать **антиградиент** функции потерь (фактически «остатки» при MSE).

**Современные реализации:**
- **XGBoost** — оптимизированный gradient boosting + регуляризация.
- **LightGBM** — быстрее на больших данных, leaf-wise growth.
- **CatBoost** — лучше с категориальными признаками, ordered boosting.

### Стекинг

- Несколько базовых моделей разных типов → их предсказания → мета-модель (часто простая, например линейная).
- Может улучшить, но требует осторожности (data leakage). Используют кросс-валидационные предсказания.

### Сравнение

| | Bagging | Boosting | Stacking |
|---|---|---|---|
| Параллельность | Да | Нет | Базовые — да |
| Зависимость моделей | Независимы | Каждая зависит от пред. | Независимы (мета — нет) |
| Снижает | Variance | Bias | Оба |
| Базовые модели | Сильные | Слабые | Разные |
| Чувствительность к параметрам | Низкая | Высокая | Средняя |

## А-13. Bias-Variance декомпозиция

### Разложение ошибки

Для регрессии с MSE:

$$E[(y - \hat f(x))^2] = \underbrace{\sigma^2}_{\text{шум}} + \underbrace{(\text{Bias}[\hat f(x)])^2}_{\text{смещение}^2} + \underbrace{\text{Var}[\hat f(x)]}_{\text{разброс}}$$

- **Шум (irreducible error):** неустранимая случайность в y. Никакая модель не уберёт.
- **Bias (смещение):** разница между средним предсказанием модели и истинной функцией. Большой bias = слишком простая модель.
- **Variance (разброс):** насколько модель меняется при смене обучающей выборки. Большой variance = слишком чувствительна к данным.

### Связь со сложностью модели

| Сложность модели | Bias | Variance | Поведение |
|---|---|---|---|
| Низкая | Высокий | Низкий | Недообучение |
| Высокая | Низкий | Высокий | Переобучение |
| Оптимальная | Низкий | Низкий | Хорошо |

**Графически:** при увеличении сложности bias² монотонно падает, variance растёт, total error имеет U-форму. Оптимум — внизу U.

### Связь с композициями

- **Бэггинг ↓ variance.** Усреднение независимых предсказаний снижает разброс. Bias почти не меняется.
- **Бустинг ↓ bias.** Каждая модель уменьшает остаточную ошибку.
- Поэтому в Random Forest базовые деревья делают **глубокими** (низкий bias, высокий variance — бэггинг подавляет variance), а в бустинге **мелкими** (низкий variance — boosting уменьшает bias).

## А-14. Кластеризация

### K-means

**Алгоритм:**
1. Выбрать K центров (часто **K-means++** — умная инициализация).
2. Каждую точку отнести к ближайшему центру (по евклидовой метрике).
3. Пересчитать центры как средние точек кластера.
4. Повторять 2-3 до сходимости.

**Цель:** минимизировать
$$\text{Inertia} = \sum_{i=1}^{K}\sum_{x \in C_i}\|x - \mu_i\|^2$$

**Плюсы:** простой, быстрый, масштабируемый.

**Минусы:**
- Нужно задавать K.
- Кластеры сферические (плохо для вытянутых).
- Чувствителен к выбросам и шуму.
- Чувствителен к масштабу признаков (нужно StandardScaler).
- Чувствителен к инициализации (запускают несколько раз).

**Выбор K:**
- **Метод локтя (elbow):** график inertia vs K, ищем «излом».
- **Silhouette score:** ищем максимум.
- **Gap statistic.**

### DBSCAN

**Параметры:**
- **ε (eps)** — радиус соседства.
- **min_samples** — минимальное число соседей в радиусе ε для core-точки.

**Типы точек:**
- **Core:** в радиусе ε есть ≥ min_samples соседей.
- **Border:** в радиусе ε core-точки, но сама не core.
- **Noise (outlier):** не достижима ни от какой core-точки.

**Плюсы:**
- Не нужно задавать число кластеров.
- Находит кластеры произвольной формы.
- Естественным образом находит выбросы.

**Минусы:**
- Плохо работает с кластерами разной плотности.
- Подбор ε и min_samples нетривиален.
- Для высокоразмерных данных страдает от «проклятия размерности».

### Иерархическая кластеризация

- **Агломеративная (снизу вверх):** каждая точка — кластер, поэтапно сливаем близкие.
- **Дивизионная (сверху вниз):** один кластер — постепенно делим.
- Результат — **дендрограмма**. Выбираем количество кластеров, обрезая на нужной высоте.
- Метрики связи (linkage): single, complete, average, ward.

### Метрики качества кластеризации

**Без истинных меток (внутренние):**
- **Silhouette score** ∈ [−1, 1]: для каждой точки a = среднее расстояние внутри своего кластера, b = до ближайшего чужого. s = (b − a)/max(a,b). Близко к 1 — отлично, 0 — на границе, отрицательное — может быть не в том кластере.
- **Davies-Bouldin:** среднее отношение разброса внутри кластера к расстоянию между кластерами. Меньше = лучше.
- **Calinski-Harabasz:** отношение между- и внутрикластерной дисперсии. Больше = лучше.
- **Inertia** (для K-means): сумма квадратов расстояний до центров.

**С истинными метками (внешние):**
- **ARI (Adjusted Rand Index):** учитывает совпадение разбиений, ∈ [−1, 1], 0 — случайное.
- **NMI (Normalized Mutual Information):** ∈ [0, 1].
- **Homogeneity, completeness, V-measure.**

## А-15. Нейронные сети

### Типы слоёв

**Dense (Fully Connected, Linear):**
- Каждый нейрон связан со всеми входами: y = Wx + b.
- Универсален, но много параметров.

**Convolutional (Conv2D):**
- Ядро (kernel) свёртки скользит по входу.
- Параметры: kernel size, stride, padding, число фильтров.
- Локальность + разделение весов → мало параметров.
- Подходит для изображений (Conv2D), сигналов (Conv1D).

**Pooling:**
- **MaxPool** — берёт максимум в окне.
- **AvgPool** — среднее.
- Уменьшает размерность, делает инвариантным к сдвигам.

**Recurrent (RNN, LSTM, GRU):**
- Для последовательностей.
- LSTM/GRU решают проблему долгих зависимостей и затухающего градиента.

**Embedding:**
- Lookup-таблица, превращает индексы в плотные векторы.

**Dropout:**
- Случайно зануляет долю p нейронов во время обучения. Регуляризация.
- На инференсе работает по-другому (или обычно вообще ничего не делает).

**Batch Normalization:**
- Нормализует активации внутри батча.
- Ускоряет обучение, помогает с градиентами.

**Attention / Self-Attention / Transformer:**
- Современные модели для последовательностей.
- Каждый токен «смотрит» на все остальные.

### Функции активации

| Функция | Формула | Диапазон | Особенности |
|---|---|---|---|
| **Identity (линейная)** | x | (−∞, ∞) | Без нелинейности. На выходе регрессии. |
| **Sigmoid** | 1/(1+e⁻ˣ) | (0, 1) | Затухающий градиент, не центрирована. Использовать на выходе бинарной классификации. |
| **Tanh** | (eˣ−e⁻ˣ)/(eˣ+e⁻ˣ) | (−1, 1) | Центрирована. Тоже затухает. |
| **ReLU** | max(0, x) | [0, ∞) | Простая, не затухает для x>0. Проблема «мёртвых нейронов» при x<0 (градиент = 0). |
| **Leaky ReLU** | max(αx, x), α≈0.01 | (−∞, ∞) | Решает мёртвые нейроны. |
| **ELU** | x при x>0, α(eˣ−1) при x≤0 | (−α, ∞) | Гладче, лучше градиенты. |
| **GELU, Swish** | x·σ(βx) и подобные | — | Используются в трансформерах. |
| **Softmax** | eˣⁱ/Σeˣʲ | (0, 1), сумма = 1 | Многоклассовая классификация (выход). |

### Какую активацию выбирать

**На выходе:**
- Регрессия → identity (линейная).
- Бинарная классификация → sigmoid.
- Многоклассовая → softmax.

**В скрытых слоях:**
- По умолчанию **ReLU**.
- Если мёртвые нейроны — Leaky ReLU, ELU.
- В трансформерах — GELU.

### Проблемы и решения

- **Vanishing gradient** (затухающий градиент): sigmoid/tanh в глубоких сетях. Решения: ReLU, BatchNorm, skip connections (ResNet).
- **Exploding gradient:** градиенты слишком большие. Решение: **gradient clipping**.
- **Overfitting:** dropout, регуляризация, аугментации, early stopping.

### Обучение

- **Forward pass:** считаем предсказание.
- **Loss:** вычисляем функцию потерь.
- **Backward pass (backpropagation):** считаем градиенты по цепному правилу.
- **Optimizer step:** обновляем веса (SGD, Adam, RMSprop).

### Типичные функции потерь для NN

- Регрессия: MSE, MAE, Huber loss.
- Бинарная классификация: Binary Cross-Entropy.
- Многоклассовая: Categorical Cross-Entropy.
